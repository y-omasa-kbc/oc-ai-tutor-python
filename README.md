## このリポジトリについて

- **正式名称**: オープンキャンパス体験授業「AI家庭教師を作ろう」演習環境
- **目的**: 高校生が 45〜50 分の体験授業で、RAG の基礎と Query Transformation による検索改善を **手を動かして学ぶ** ための環境
- **対象ユーザー**:
  - **受講者**(高校生・PC 初心者): 授業中にコードを改造する
  - **授業者(教員)**: 授業準備・当日のトラブル対応
- **扱える題材**: 日本史のある一時期に関する RAG(データ差し替えで他分野にも転用可)
- **配布形態**: 授業用 PC にリポジトリを配置し、受講者 1 人 1 台の DevContainer を提供

---

## 前提条件

受講者 PC に **以下がインストール済み** であることを確認してください(授業前のセットアップで必須)。

| ソフトウェア | バージョン目安 | 備考 |
|--------------|----------------|------|
| **Docker Desktop** | 最新版 | WSL2 バックエンドを有効化(Windows) |
| **Visual Studio Code** | 最新版 | |
| **VSCode 拡張: Dev Containers** | 最新版 | Microsoft 製。ID: `ms-vscode-remote.remote-containers` |
| ブラウザ | Edge / Chrome / Firefox のいずれか | 動作確認用 Web UI を開きます |
| Git | 任意(clone に使う場合) | VSCode の GUI クローンでも可 |

> **受講者 PC で上記が揃っていない** 場合は授業前にセットアップを完了させてください。**授業時間内での環境構築は想定していません**。

### ネットワーク要件

- 授業中、**OpenRouter**(`https://openrouter.ai/`)への **HTTPS 接続** が必要です。
- 本校の **プロキシ・ファイアウォール** 設定によっては、OpenRouter ドメインが許可されているか事前確認してください。

## セットアップ手順

### 1. リポジトリを取得する

#### 初めてセットアップする場合(授業前日 / 当日の朝)

```bash
# 任意の作業ディレクトリで
git clone <このリポジトリの URL> oc-ai-tutor-python
cd oc-ai-tutor-python
```

#### 既にクローン済みで、最新にしたい場合

```bash
cd oc-ai-tutor-python
git pull
```

> 受講者には **Git 操作を直接させません**。授業者が事前にクローンしておくか、VSCode の **「ソース管理」** からクローンするよう案内します。

---

### 2. `.env` を作成する

**API キーは絶対にコミットしないでください**。このリポジトリでは `.env` を `.gitignore` で除外しています。

#### 2-1. 雛形をコピーする

```bash
cp .env.example .env
```

#### 2-2. `.env` を編集する

エディタで `.env` を開き、以下を設定します。

```ini
# OpenRouter の API キー(必須)
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxxxxxxxxxxxxxxxxxx

# 受講者の気兼ねを減らすため、ダミーのキー名でも可
# OPENROUTER_API_KEY=sk-or-v1-ここに本物のキーを入れる

# その他、必要に応じて(後述の「利用する環境変数」参照)
# OPENROUTER_MODEL=...
# OPENROUTER_EMBEDDING_MODEL=...
```

**絶対にやってはいけないこと**:
- `.env` を **Git にコミット** する
- `.env` を **スクリーンショットに撮る** / **SNS に貼る**
- `.env` を **受講者に書き換えさせる**

#### 2-3. 授業用 PC 一括配布(教員向け)

授業前、授業用 PC の台数分 `.env` を **USB メモリまたは本校ファイル共有サーバー** 経由で配置します。

| 配布方法 | 手順 | 配布後の確認 |
|----------|------|--------------|
| USB メモリ | 1 本の USB に `.env` を入れ、PC 1 台ずつ挿してコピー | 各 PC で `.env` がリポジトリ直下にあることを確認 |
| ファイル共有 | 共有フォルダに配置し、各 PC からコピー | 同上。**授業後に共有フォルダから必ず削除** |

> 詳細は [`instruction.md` の「3-4. T-30(授業 30 分前)」](../instruction.md) を参照してください。

---

### 3. DevContainer で開く

1. VSCode で **本リポジトリのフォルダ** を開く
2. 右下に **「コンテナーで再度開く」** というポップアップが出るのでクリック
   - 出ない場合は、コマンドパレット(`Ctrl+Shift+P` / `Cmd+Shift+P`)で **「Dev Containers: Reopen in Container」** を実行
3. **初回はビルドに数分かかります**(ネットワーク速度と PC 性能に依存)
4. ビルドが完了すると、左下に **「Dev Container: ...」** と表示される

> **ビルドが失敗** する場合は[トラブルシューティング](#トラブルシューティング)を参照してください。

---

### 4. 動作確認

1. **動作確認用 Web UI** をブラウザで開く
   - VSCode の「ポート」タブ(`Ctrl+Shift+P` → `Ports`)で `8501`(Streamlit の場合)を確認し、ブラウザで開く
   - または VSCode 右下に **「ブラウザで開く」** 通知が出るのでそれをクリック
2. テキストボックスに例示の質問を入力し、送信
3. 回答と **参考資料** が表示されれば成功

**期待される初期状態**:
- **単純 RAG**(Query Transformation なし)が動作している
- 授業の途中で受講者が Query Transformation を組み込む

## 日常の操作

### アプリを起動する

DevContainer 起動時に **自動でアプリが立ち上がる** 設計です。立ち上がらない場合は、VSCode の **端末(Terminal)** で以下を実行:

```bash
# (実装によってコマンドは異なります。確定後、本 README を更新)
# 例:
# docker compose up -d
# streamlit run app/web/app.py
```

### アプリを停止する

- DevContainer を閉じる(`Dev Containers: Close Remote Connection`)
- あるいは VSCode の **「ファイル > コンテナーを閉じる」**

### コードを変更する

- 受講者が VSCode で **Python ファイル**(`app/rag/` 配下)を編集する
- アプリは **ホットリロード** 対応が望ましい(実装時に確認)
- 反映されない場合は[トラブルシューティング](#トラブルシューティング)を参照

### コンテナをビルドし直す

依存パッケージを追加した / `Dockerfile` を変更した 場合は再ビルド:

```bash
# VSCode コマンドパレットから:
# Dev Containers: Rebuild Container
```

```bash
# あるいは docker compose で:
docker compose build --no-cache
docker compose up -d
```

## API キーの扱い

### 基本方針

| 環境 | キー管理 | 配布経路 | 受講者の扱い |
|------|----------|----------|--------------|
| 体験授業(このリポジトリ) | `.env`(リポジトリ直下) | **教員が USB / ファイル共有で配置** | **受講者は触らない** |
| フォローアップ(Web アプリ) | サーバー側環境変数 | 教員が事前に設定 | 受講者は触らない |

### 受講者に API キーの設定を **絶対に** させない

- 受講者による `.env` 編集は **想定外のエラーと漏洩リスク** を生む
- 授業中に「API キーを打ち込んで」と言われた時点で **何かが間違っている**
- 授業前の `.env` 配置が抜けていた場合は、**補助が対応**

### キーが漏れた疑いがある場合

1. **直ちに OpenRouter のダッシュボードで該当キーを revoke**
2. 新規キーを発行し、`.env` を再配置
3. インシデントとして [`instruction.md` の「振り返り」](../instruction.md) に記録

## トラブルシューティング

### DevContainer のビルドに失敗する

| 症状 | 原因候補 | 対応 |
|------|----------|------|
| ビルドが途中で止まる | ネットワーク不通 / Docker Hub に繋がらない | プロキシ・ファイアウォールを確認 |
| `apt-get` 関連でエラー | Docker キャッシュの破損 | 「Dev Containers: Rebuild Container」(キャッシュなし) |
| メモリ不足でビルド失敗 | Docker Desktop のリソース不足 | Docker Desktop の設定でメモリ割当を増やす(4GB 以上推奨) |
| ポート競合 | 既に別プロセスがポート使用 | 該当プロセスを停止 / `docker-compose.yml` のポートを変更 |

### アプリが起動しない

1. VSCode の **「ポート」** タブで対象ポートが開いているか確認
2. 端末で `docker compose ps` を実行し、コンテナが `Up` か確認
3. `docker compose logs -f` でログを確認
4. `.env` が **正しい場所**(リポジトリ直下)にあるか確認

### コードを変更しても反映されない

1. アプリが **ホットリロード** に対応しているか確認(対応状況は実装に依存)
2. ブラウザを **ハードリロード**(`Ctrl+Shift+R` / `Cmd+Shift+R`)
3. VSCode の **「ファイル > コンテナーをリスタート」** を試す

### API キーが認識されない(401/403 エラー)

1. `.env` が **リポジトリ直下** にあるか確認
2. 値の **前後** にスペース・改行が入っていないか確認
3. OpenRouter のダッシュボードで **キーが有効** か確認
4. レートリミットに達していないか確認

### それでも動かないとき

- VSCode の **「Dev Containers: Rebuild Container」** を実行
- それでもダメなら、**Docker Desktop を完全再起動**
- 最終手段: **受講者を別 PC と交代** させ、補助が復旧を継続

## 教員向けの補足

### 授業の準備・リハーサル

- 授業 1 週間前 / 3 日前 / 1 日前 / 30 分前 のチェックリストは [`instruction.md` の「3. 事前準備チェックリスト」](../instruction.md) を参照
- **リハーサルは必ず同じ PC・同じネットワーク** で実施する
- 授業中に見せたい **失敗質問・成功質問** をリハーサルで確定させておく

### 授業後: 受講者の変更をリセットする

受講者が編集した Python ファイル(主に `app/rag/query_transformer.py` と `app/rag/prompts.py`)は、**次回授業のために初期状態へ戻します**。受講者の作業結果は保管しません。

**手順**(コンテナ内ターミナル または ホスト端末で):

```bash
cd /workspace           # DevContainer 内。ホストの場合はリポジトリのパスへ
git reset --hard HEAD
```

**絶対にやってはいけないこと**:

- **`git commit` を実行する** ― 受講者の作業を誤ってコミットしてしまうと、`git reset --hard HEAD` で初期状態に戻せなくなります。受講者と補助者に「commit / push は行わない」を徹底してください。
- **`git push` を試みる** ― 受講者 PC からリモートリポジトリへ書き戻す運用はありません。

`.env` は `.gitignore` で除外されているため、リセットの影響を受けません。ファイル共有で配布した `.env` は共有元から削除、USB 配布分は回収してください。

### 授業後の振り返り

- 授業後 **24 時間以内** に [`instruction.md` の「10. 授業後の後処理と振り返り」](../instruction.md) を実施
- トラブル事例・受講者の反応は **本リポジトリの Issue ではなく `instruction.md` に追記**(本リポジトリは受講者用公開のため、内部反省は upstream の `prj-oc-ai-tutor-python` に集約)

### このリポジトリを更新する際の注意

- **`.env` が混入していないか** 必ず `git status` で確認
- 機密情報(API キー、生徒情報、学校固有の URL 等)を **絶対にコミットしない**
- 大きな変更(Dockerfile 変更、依存追加等)は **授業前に余裕をもってリハーサル**
