1. はじめに

今日は 45 分で、AI の中身をちょっとだけ改造する 体験をします。 プログラミング経験はゼロで OK。コピペ中心 なので、PC の操作はそこまで難しくありません。

ゴールは 3 つ:

  • RAG(検索 + 回答生成)の動きを 自分の目で確認 する
  • 質問の書き換え(Query Transformation)で 検索の精度が変わる ことを体感する
  • AI への指示(プロンプト)の書き方で 回答の雰囲気や内容が変わる ことを体感する

2. RAG とは?

RAG は Retrieval-Augmented Generation の略で、日本語にすると「検索して、答えを作らせる」手法です。

次の 4 つのステップで動きます:

  1. ユーザーが 質問 を入力する
  2. AI が 資料 の中から、質問に合いそうなものを 検索 する
  3. 見つけた資料を LLM に渡す
  4. LLM が資料を読みながら 回答 を作る

普通の ChatGPT と違うのは、資料を渡してから答えさせる ところです。 今日はこの「資料を渡すところ」を、実際に 江戸時代の歴史 でやってみます。

3. 単純 RAG を試す

いま右側に出ている Web UI(画面に表示されているアプリ)が 単純 RAG です。 質問を入れると、AI が資料(「秀吉の天下統一」「関ヶ原への道」「江戸幕府の確立」)から答えを探して答えてくれます。

やってみよう:

  1. 隣のウィンドウでアプリを表示
  2. 例: 本能寺の変を起こした人物は? と入力して送信
  3. 「参考にした資料」の欄に どの資料が引かれたか が出るので確認

質問の 言葉そのまま で資料を探すので、質問の言い回し が大事、というのがポイントです。

4. 検索の落とし穴

質問が 長すぎたり、抽象的だったり すると、思った資料が引かれません。

失敗しやすい質問の例(講師のデモを観察):

  • 「信長の後継者争いの結果、最終的に誰が天下人になった?」
  • 「家康が天下を取った経緯をざっくり教えて」

これらは 質問の言葉 と 資料の中の言葉 がズレているため、 単純 RAG だと まったく違う資料 が引かれたり、回答がぼんやりしたりします。

そこで次は、AI に 質問を作り直させる 機能を追加します。

5. 質問改善(Query Transformation)を実装する

Query Transformation とは、AI に「この人、こういう意味で聞いてるな」と解釈させて、 検索しやすい 別の言い回し に変換してから資料を探すテクニックです。

具体的には、LLM に「曖昧な検索クエリを、明確で簡潔な検索クエリに書き直して」とお願いする関数を 1 個入れます。

手順

  1. VSCode で app/rag/query_transformer.py を開く
  2. 下の コピー ボタンで サンプル全体 をコピー
  3. VSCode に戻って Ctrl+A で全選択 → Delete で中身を消す
  4. コピーしたサンプルを Ctrl+V で貼り付け
  5. Ctrl+S で保存(右側の Web UI は自動でリロードされます)

コピーするコード

下の 「コピー」ボタン を押してください。コードがクリップボードにコピーされます。 

import os

from app.rag.llm import chat

QUERY_REWRITE_SYSTEM_PROMPT = (
    "あなたは日本語の検索クエリを、検索エンジンやベクトル検索にかけやすい形に整えるアシスタントです。"
    "ユーザーの質問が曖昧だったり長かったりする場合は、"
    "元の意図を保ったまま、具体的で簡潔な検索クエリに書き直してください。"
    "質問がすでに明確で短い場合は、そのまま返してください。"
    "余計な説明はせず、書き換えたクエリだけを出力してください。"
)


def transform_query(query: str) -> str:
    if not query.strip():
        return query

    if os.getenv("DISABLE_QUERY_TRANSFORM", "0") == "1":
        return query

    messages = [
        {"role": "system", "content": QUERY_REWRITE_SYSTEM_PROMPT},
        {"role": "user", "content": query},
    ]
    try:
        rewritten = chat(messages=messages, temperature=0.0)
        return rewritten.strip() or query
    except Exception:
        return query
確認してみよう
Web UI の左サイドバーで 「質問改善 (Query Transformation)」 を ON にして、 さっき失敗した質問をもう一度入れてみてください。引かれる資料や回答が 変わる はずです。

6. システムプロンプトを変える

ここまでは「資料を探す」側を変えてきました。 ここでは「回答を作る」側の AI への指示、システムプロンプト をいじってみます。

同じ資料と質問でも、指示の書き方 で回答の雰囲気や内容が変わります。

手順

  1. VSCode で app/rag/prompts.py を開く
  2. 下の コピー ボタンで、好みの 雛形サンプル をコピー
  3. SYSTEM_PROMPT の中身を貼り付けて 1 箇所だけ書き換え(★コメントを目印に)
  4. 保存 → Web UI で同じ質問を実行

雛形サンプル A: もっとカジュアルに

堅い歴史の文章を、友達に説明するような口調にしたいとき。

SYSTEM_PROMPT = (
    "あなたは江戸時代の歴史に詳しい、友達みたいな『歴史の家庭教師』です。"
    "ユーザーからの質問に対して、与えられた参考資料だけを根拠に、"
    "★中学生にもわかるよう、タメ口で、親しみやすく答えてください。"
    "回答は3〜6文程度の短さを心がけ、結論→理由→具体例 の順で話してください。"
    "参考資料に書かれていないことは推測せず、"
    "分からない場合は『ごめん、参考資料からは分からんわ』と正直に答えてください。"
)

雛形サンプル B: 出典を必ず示す

出典の明示を徹底したいとき。用語集・レポート向け。

SYSTEM_PROMPT = (
    "あなたは江戸時代の歴史に詳しい、厳密な『歴史研究助手』です。"
    "ユーザーからの質問に対して、与えられた参考資料だけを根拠に、"
    "★回答の末尾に必ず [資料: ファイル名] の形で出典を明記してください。"
    "回答は3〜6文程度の短さを心がけ、結論→理由→具体例 の順で話してください。"
    "参考資料に書かれていないことは推測せず、"
    "分からない場合は『参考資料からは分かりません』と正直に答えてください。"
)

雛形サンプル C: 初心者向けに噛み砕く

歴史をほぼ知らない相手に教える場面。

SYSTEM_PROMPT = (
    "あなたは江戸時代の歴史に詳しい、はじめての人に寄り添う『歴史の家庭教師』です。"
    "ユーザーからの質問に対して、与えられた参考資料だけを根拠に、"
    "★中学1年生にもわかる言葉で、むずかしい漢字は開き直さず、易しい言葉に置き換えて答えてください。"
    "回答は3〜6文程度の短さを心がけ、結論→理由→具体例 の順で話してください。"
    "参考資料に書かれていないことは推測せず、"
    "分からない場合は『参考資料からは分かりません』と正直に答えてください。"
)
★ここを変える
雛形の中の ★ で始まる 1 文を 別の指示 に書き換えてみてください。 例: 「語尾を『である調」に」「100字以内で要約して」「受験生の質問に答える気持ちで」

7. まとめ

今日やったことを 3 行で:

  • RAG の検索は質問の言い回しに弱い → 質問が長い・抽象的だと資料を外す
  • 質問を作り直させると改善する → Query Transformation で検索ヒットが安定する
  • 指示の書き方で回答が変わる → システムプロンプト 1 行で AI のキャラクターが動く
